## Day Detail Timeline / Progress & Status Logging
- Backend: PUT /api/flows/:flowId/tasks/:id (server.js ~line 1551)

  - statusLog: logs { from, to, at, by } when status changes
- Frontend: saveDetail() sends data.fromProgress = prevProgress alongside progress
- Frontend: openDayDetail() builds timeline from createdAt, updatedAt, statusLog, progressLog, notes
  - Timeline sorts newest first (descending)
  - Entry types: created, status, updated, note, progress
  - Each entry shows priority dot + task title + action label + detail content
  - Progress entries show change banner with from→to bar visualization
  - Status entries show from→to colored badges

## Data Model Notes
- Tasks: lastActivityAt field (ISO string) — updated on status/progress/assignment changes and note additions
- Tasks: recurrenceRule object — {type: daily|weekdays|weekly|monthly|custom, interval, endType: never|date|count, endDate, endCount}
- Tasks: recurrenceSeriesId — UUID linking a recurring series, _recurrenceSpawned flag prevents double-spawn
- Users: availabilityBlocks array (each: {id, from, to, reason, type, createdAt})
- Config: staleThresholds (minutes per status), reminderHours (array of hours before deadline, default [24, 48])
- Notification types: task_stale, deadline_reminder (in addition to existing types)
- Templates stored in templates.json, auto-created on first read

## API Key System (COMPLETED)
- Auth: authMiddleware updated (async), checks tfk_ prefix → apikeys.json lookup, else → sessions Map
- API key format: tfk_ + 48 random hex chars (crypto.randomBytes(24))
- Storage: data/apikeys.json — [{id, userId, userName, name, key, createdAt, lastUsedAt, revoked}]
